# Policies (Admins and Authors)

## Create a policy

1) Open `Policies`.
2) Select `New Policy`.
3) Choose a template and fill in required fields.
4) Review the generated content and edit if needed.
5) Save the policy as a draft.

## Policy metadata

When creating or editing, you can set:

- Policy name
- Framework and mapped frameworks
- Business unit
- Review frequency and justification
- Recompliance or next review date

## Submit for approval

1) Open a draft policy.
2) Select `Submit for Review`.
3) Add notes if required.
4) The policy moves into the approval queue.

## Edit or update a policy

1) Open the policy.
2) Select `Edit` and apply changes.
3) Save to create a new version.

## Version history

- Each save creates a new version.
- The latest version is used for approvals and acknowledgements.

## Archive or restore

- Use `Archive` to remove outdated policies from active lists.
- Use `Restore` to bring a policy back to active status.

## Batch actions (if enabled)

Some views support batch actions such as approve, archive, delete, or restore multiple policies at once.
